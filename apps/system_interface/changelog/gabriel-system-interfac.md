Revealing a merged system-interface change is now an explicit step the agent runs, scoped to what changed. A frontend change is rebuilt with `npm run build` and revealed in place by `reload_system_interface.py`, which broadcasts a `reload_system_interface` op so every open browser reloads the whole top-level page -- no service restart needed. A backend change is revealed by restarting the service. The previous auto-reveal machinery (the conditional-rebuild service step, the `/api/build-id` endpoint and build-id meta tag, and the websocket-reconnect build-id check) is gone.
