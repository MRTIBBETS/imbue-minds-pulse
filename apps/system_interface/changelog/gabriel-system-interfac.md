Revealing a system-interface change after merge is now a single command, `mngr start --restart system-services`. On startup the service rebuilds the frontend bundle only when its sources changed, and any open browser reloads itself into the new bundle on reconnect (via a build-id check the backend exposes at `/api/build-id`). The separate `npm run build` + manual full-page-reload step (and the `reload_interface` broadcast op) are gone.
